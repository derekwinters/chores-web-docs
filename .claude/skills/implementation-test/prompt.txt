Run the strict docs build for implemented changes.

No input required. Runs in current git branch. This repo has no test suite;
`mkdocs build --strict` is the PR gate.

Steps:
1. Run the strict build: mkdocs build --strict
2. Capture output: warnings-as-errors, first failing message
3. Check exit code: 0 = success, non-zero = failure
4. Report results:
   - If PASS: Strict build passed. Ready for verification.
   - If FAIL: List the strict-mode failures and error messages. Blocks workflow.

Output format - Success:
✅ Strict build passed
- mkdocs build --strict: clean
- Status: Ready for verification phase

Output format - Failure:
❌ Strict build failed
- mkdocs build --strict reported errors:
  - WARNING - A relative path to 'guide/missing.md' is included in the nav ...
  - WARNING - Doc file 'guide/index.md' contains a link '../gone.md' ...
- Status: Blocked - fix strict failures and retry

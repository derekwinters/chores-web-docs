# Release v1.8.0

v1.8.0 improves error visibility, adds a configurable overdue-chore schedule setting, and fixes several UI consistency issues across mobile and settings pages.

<!-- more -->

## Features

### Chore Settings

- Overdue-chore transition hour is now configurable — set the hour of day when scheduled chores become overdue via the new `due_time_hour` setting (#205)

### Error Handling

- Structured error handling added across backend routers and services — errors are logged with proper HTTP status codes instead of silently failing (#195)

## Bug Fixes

- Hidden backend errors now surface in the Log page UI — fetch failures show an error message instead of a blank page (#204)
- Mobile top navigation dropdown now opens downward — the UserAvatarMenu no longer clips off-screen on small viewports (#216)
- Settings sidebar HR now aligns with the heading and nav links — consistent left-edge alignment across all sidebar elements (#199)
- Sidebar state is restored on viewport resize — the sidebar remembers your desktop preference and auto-closes on mobile instead of getting stuck (#181)

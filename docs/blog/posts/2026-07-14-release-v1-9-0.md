# Release v1.9.0

v1.9.0 introduces the notification system: Chores Web now tells each person when
their chores become due — in the web app through an in-app bell and toast, and on
Android through system notifications — with per-type preferences and an offline
alert on mobile.

<!-- more -->

## Features

### Notifications

- Chore-due notifications — when a chore becomes due, Chores Web generates a
  notification for each person the chore is relevant to, at the household's
  configured due-time transition
- Server-owned delivery, acknowledgement, and automatic dismissal — a
  notification is delivered on first fetch, stays consistent across devices, and
  is cleared automatically once its chore is no longer due, so there is nothing
  to clean up by hand

### Web App

- In-app notification surfacing — a top-bar bell with an unread badge, a toast
  when new notifications arrive, and a notification log where items can be
  acknowledged

### Android App

- System notifications via background polling — the Android app polls the server
  with WorkManager and posts due chores as native system notifications (no
  push/FCM required)
- Configurable poll interval — tune how often the app checks for new
  notifications from the app's settings
- "Not connected in X days" alert — a local reminder that fires when the app
  hasn't reached the server recently, with a configurable threshold

### Preferences

- Per-type notification preferences — enable or disable the chore-due
  notification type from either the web or Android settings; types are enabled by
  default

See the [Notifications guide](../../user-guide/notifications.md) for full details.
